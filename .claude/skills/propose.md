# Propose Skill

現在のプロジェクトのコード・UI・UX・技術スタックを読み込んで、「やったら良さそうなこと」を提案する。

## 実行手順

1. CLAUDE.md・主要なソースファイル・package.json を読み込んでプロジェクトの全体像を把握する
2. 以下の観点から改善案・新機能案を洗い出す
3. 実装コストと効果を見積もって優先度をつけて提案する

## 観点リスト

### UX・操作性
- 操作ステップを減らせる箇所（ワンタップでできそうなこと）
- フィードバックが弱い箇所（ローディング表示・完了通知・エラー表示）
- よく使う操作へのショートカット不足
- 初めて使うユーザーが迷いそうな箇所（オンボーディング・ツールチップ）

### モバイル特有
- スワイプジェスチャーで代替できる操作
- キーボード表示時のレイアウト崩れ対策
- 長押し・ピンチなどのネイティブライクな操作
- オフライン時の動作・再接続フロー

### 機能追加
- ユーザーから「あったら便利」と言われそうな機能
- 既存機能を組み合わせて実現できる便利な機能
- 他の類似ツールが持っていてこのツールにない機能

### パフォーマンス
- 不要な再レンダリングが起きている箇所
- 画像・リソースの最適化余地
- WebSocket通信の効率化余地
- キャッシュ・メモ化が有効な箇所

### 技術的負債・保守性
- 重複しているロジック・コンポーネント
- ハードコードされた値で設定化できるもの
- エラーハンドリングが甘い箇所
- テストが書けそうな重要なロジック

### 堅牢性・安全性
- 想定外の入力・状態でクラッシュしそうな箇所
- セッション切断・再接続の考慮漏れ
- データの永続化・バックアップが必要な箇所

## 提案フォーマット

優先度 high / medium / low に分けて列挙する。
各提案は以下の形式で記載する：

```
### [優先度] タイトル
**概要**: 何をするか1〜2文で
**なぜ良いか**: ユーザーへの具体的なメリット
**実装イメージ**: どのファイルをどう変えるか（大まかで良い）
**コスト**: 小 / 中 / 大
```

最後に「今すぐやるなら TOP 3」を一言でまとめる。
