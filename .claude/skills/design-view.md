# Design View Skill

UIデザインの品質を体系的にレビューする。以下の観点でCSSとコンポーネントファイルを読み込み、問題点を洗い出して報告する。

## 実行手順

1. 対象のCSSファイルとJSX/TSXコンポーネントをすべて読み込む
2. 以下のチェックリストを順番に確認する
3. 問題が見つかったら `ファイル名:行番号` 形式で列挙し、修正案を添える
4. 問題がなければ「✓ 問題なし」と記載する

## チェックリスト

### 1. フォント統一性
- font-family がハードコードされて混在していないか（var(--font) などのCSS変数を使っているか）
- font-size が任意の値で散乱していないか（一定のスケール 11/12/13/14/16px 程度に収まっているか）
- font-weight の使い方が一貫しているか（700=太字、400=通常 など）
- line-height が指定されていない箇所でレイアウトが崩れていないか

### 2. スペーシング・マージン
- padding/margin がマジックナンバーになっていないか（4の倍数グリッドが望ましい）
- 同じ役割の要素で padding/gap の値が揃っているか（ボタン同士、カード同士など）
- ネストが深い要素で余白が二重に効いて詰まりすぎ/広すぎになっていないか
- セーフエリア（iOS底部のノッチ）対応が漏れていないか（padding-bottom: env(safe-area-inset-bottom)）

### 3. カラーシステム
- ハードコードされた色（#xxxxxx）がCSS変数（--accent, --text, --bg など）で置き換えられるのに放置されていないか
- 同じ目的の色が複数の値で書かれていないか（例：黒が #000, #0a0f0d, #111 と混在）
- テーマ変更（アクセントカラー切替）に追従しない箇所がないか

### 4. ホバー・フォーカス・アクティブ状態
- ボタンのホバー色が theme と無関係にハードコードされていないか
- disabled 状態のスタイルが明確に区別されているか（opacity / pointer-events）
- フォーカスリング（outline）が消されたままになっていないか（キーボード操作への影響）
- タッチデバイスで :hover が残留して見た目が壊れる箇所がないか

### 5. レスポンシブ・ブレークポイント
- モバイル用コンポーネントとPC用コンポーネントで同じ情報の表示方法が矛盾していないか
- メディアクエリの境界値が一貫しているか（1024px を境に切替など）
- フォントサイズ/ボタンサイズがモバイルで小さすぎ（最低44px タップターゲット）になっていないか
- ランドスケープ（横向き）時に要素がはみ出したり消えたりしていないか

### 6. テキスト折り返し・オーバーフロー
- 長いテキスト（セッション名・セリフ・コマンド）が折り返しで崩れる箇所がないか
- overflow: hidden で切り捨てるなら text-overflow: ellipsis が必要な箇所に漏れがないか
- white-space: nowrap が使われている箇所でコンテナからはみ出していないか
- word-break / overflow-wrap の指定漏れがないか

### 7. z-index 管理
- z-index がハードコードの大きな値（9999など）で競合している箇所がないか
- モーダル・ドロワー・ポップアップの重なり順が意図通りか
- fixed/sticky 要素が他の要素に隠れる/覆いかぶさる問題がないか

### 8. アニメーション・トランジション
- transition が付いていないのにカクっと切り替わるインタラクションがないか
- アニメーション時間がバラバラ（0.1s, 0.3s, 500ms が混在）になっていないか
- prefers-reduced-motion への配慮が必要な派手なアニメーションがないか

### 9. アイコン・画像
- img に width/height または aspect-ratio が指定されており、レイアウトシフトが起きないか
- object-fit の指定でトリミング意図が明確になっているか
- alt テキストが適切か（装飾画像なら alt=""）

### 10. CSS セレクタ・特異性
- !important が多用されていないか（あれば場所と理由を確認）
- インラインスタイル（style={{ }}）とCSSクラスが混在して優先順位が複雑になっていないか
- 未使用のCSSクラスが残っていないか

## 報告フォーマット

各チェック項目について以下の形式で報告する：

```
## [チェック項目名]
**問題なし** または

**[severity: high/medium/low]** `ファイル:行番号`
> 問題の説明
> 修正案: ...
```

最後に「総評」として全体的な品質評価と優先度の高い修正3件をまとめる。
